## Notes

- **Rate limiter**: Auth uses DB-backed `checkDbRateLimit` (persists across restarts). Search uses in-memory `checkRateLimit` (fast, resets on restart). Falls back to in-memory if DB is unreachable.
- **Logging**: Structured JSON output in production (`console.log`), readable format in development. Designed for ingestion by log services (CloudWatch, Datadog, etc.).
- **Error reporting**: `reportError()` wrapper logs structured error data. `withErrorLogging()` HOF wraps sync/async functions for automatic error capture.
- **Proxy**: Single `proxy.ts` at project root consolidates auth protection (NextAuth JWT) and security headers (`X-Robots-Tag`, `X-Auth-RateLimit`). Matches all non-static routes except API routes.
- **Image domains**: Removed wildcard `**` hostname. Only `images.unsplash.com` and `*.cloudinary.com` allowed.
- **Security headers**: `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, `X-XSS-Protection: 1; mode=block`, `Referrer-Policy: strict-origin-when-cross-origin`.
- **PWA**: Service worker generated on build via `@ducanh2912/next-pwa`. Verify `public/sw.js` and `public/workbox-*.js` exist after build.
- **RateLimit DB model**: Added to Prisma schema. Health check endpoint (`/api/health`) automatically sweeps expired rate limit entries. Requires migration: run `npx prisma db push` (or `prisma migrate dev`) to sync.
